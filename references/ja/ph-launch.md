# Product Huntローンチ

## タイミング考慮事項

| 要因 | 推奨 |
|:---|:---|
| 製品成熟度 | トラフィック急増に対応できる安定性を確保 |
| コミュニティ基盤 | Discord/Telegramコミュニティを準備 |
| GitHub Stars | PHローンチ前に500+を推奨 |
| 競争 | 有名製品と同日ローンチを避ける |
| 曜日 | 火-木がベスト；月曜は競争激しい、週末は低トラフィック |

## PHページアセット

| アセット | 要件 |
|:---|:---|
| タグライン | 一言説明、ウェブサイトと一致 |

## アップボート戦略

> ⚠️ **重要:** Product Huntは人為的な投票操作を厳しく禁止しています。すべてのアップボートは、実際に製品を使用したか、真に興味を持つユーザーから来る必要があります。

### 真正なアップボートを得る方法

| チャンネル | ベストプラクティス |
|:---|:---|
| 既存コミュニティ | Discord/Telegram/メール登録者に通知 |
| ソーシャルメディア | Twitter/LinkedInで誠実なメッセージと共にPHページを共有 |
| 個人ネットワーク | 製品を実際に使用した友人や同僚に個別に連絡 |
| メールリスト | ランチ当日に登録者にメールを送信し、PHページへのリンクを添付 |

### 紧急対応プラン
| 状況 | 対応 |
|:---|:---|
| Featuredされない | support@producthunt.comに連絡 |
| 他者に先にローンチされた | PHに連絡、所有権証明を提供 |
| 投票が削除された | 不自然な活動がないかプロモーション戦略を見直し、真正なコミュニティエンゲージメントに注力 |
