# Product Hunt 런칭

## 타이밍 고려사항

| 요소 | 권장 |
|:---|:---|
| 제품 성숙도 | 트래픽 급증에 대비한 안정성 확보 |
| 커뮤니티 기반 | Discord/Telegram 커뮤니티 준비 |
| GitHub Stars | PH 런칭 전 500+ 권장 |
| 경쟁 | 유명 제품과 같은 날 런칭 피하기 |
| 요일 | 화-목이 최적; 월요일은 경쟁 치열, 주말은 낮은 트래픽 |

---

## 헌터 선택

| 유형 | 장점 | 단점 |
|:---|:---|:---|
| **탑 헌터** | 팔로워 많음, 강한 추천 | 비쌈, 일정 빡빡 |
| **중간급 헌터** | 가성비 좋음, 원활한 소통 | 영향력 적음 |

## PH 페이지 자산

| 자산 | 요구사항 |
|:---|:---|
| 태그라인 | 한 줄 설명, 웹사이트와 일치 |
| 설명 | 상세한 가치 제안, 차별화 강조 |
| 5개 스크린샷/GIF | 각 핵심 판매 포인트에 대응 |
| 브랜드 비디오 | 첫 번째 위치, 사용자의 첫인상 |

## 업보트 전략

> ⚠️ **중요:** Product Hunt은 인위적인 투표 조작을 엄격히 금지합니다. 모든 업보트는 실제로 제품을 사용했거나 진정으로 관심 있는 사용자로부터 와야 합니다.

### 진정한 업보트를 얻는 방법

| 채널 | 베스트 프랙티스 |
|:---|:---|
| 기존 커뮤니티 | Discord/Telegram/이메일 구독자에게 알림 |
| 소셜 미디어 | Twitter/LinkedIn에서 진심 어린 메시지와 함께 PH 페이지 공유 |
| 개인 네트워크 | 실제로 제품을 사용한 친구, 동료, 어드바이저에게 개별 연락 |
| 이메일 리스트 | 런치 당일 구독자에게 이메일 전송, PH 페이지 링크 첨부 |

### 비상 대응 계획
| 상황 | 대응 |
|:---|:---|
| Featured 안됨 | support@producthunt.com 연락 |
| 타인이 먼저 런칭 | PH에 연락, 소유권 증명 제공 |
| 투표가 삭제됨 | 부자연스러운 활동이 없는지 프로모션 전략을 검토하고 진정한 커뮤니티 에마게이지먼트에 집중 |
